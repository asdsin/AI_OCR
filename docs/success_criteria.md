# 성공 기준 문서

> 파일럿 시스템의 품질 목표와 합격 기준을 정의합니다.

---

## 1. 판정 성공률

| 항목 | 목표 | 측정 방법 | 비고 |
|------|------|----------|------|
| 수치형 OCR 인식률 | **95% 이상** | (정확 인식 건수 / 전체 인식 건수) × 100 | 자릿수+소수점 포맷 필터 적용 후 기준 |
| 수치형 판정 정확도 | **98% 이상** | (정확 판정 건수 / 전체 판정 건수) × 100 | 인식 후 범위 비교 결과 기준 |
| 신호형 판정 정확도 | **99% 이상** | OK/NG 키워드 매칭 결과 기준 | 키워드가 명확하므로 높은 목표 |
| 색상형 판정 정확도 | **90% 이상** | 색상 분류 결과 기준 | 촬영 환경(조명)에 따라 편차 |

### 정확도 산출 기준
- **정확 인식**: OCR이 읽은 값이 실제 PLC 표시값과 동일 (자릿수 포맷 적용 후)
- **정확 판정**: 인식된 값에 대한 OK/NG 판정이 사람의 판정과 동일
- **인식률과 판정 정확도는 별도 측정** (인식이 틀려도 범위 안에 있으면 판정은 맞을 수 있음)

## 2. 처리 시간

| 항목 | 목표 | 비고 |
|------|------|------|
| QR 스캔 → 설비 식별 | **2초 이내** | jsQR 실시간 스캔 |
| 사진 촬영 → OCR 완료 | **5초 이내** | Tesseract.js 브라우저 실행 기준 |
| 단일 영역 판정 | **3초 이내** | 크롭 + OCR + 포맷 필터 + 범위 비교 |
| 전체 판정 (4영역) | **15초 이내** | 영역 순차 처리 |
| 결과 저장 | **1초 이내** | SQLite INSERT |

### 1건 처리 전체 플로우 목표: **QR 스캔부터 결과 확인까지 30초 이내**

## 3. 재촬영 허용 기준

| 상황 | 허용 여부 | 조치 |
|------|---------|------|
| OCR 인식 실패 (값 없음) | ✅ 재촬영 권장 | "인식 실패 — 다시 촬영하세요" 안내 |
| OCR 신뢰도 50% 미만 | ✅ 재촬영 권장 | "인식 불확실 — 확인 후 재촬영 권장" 안내 |
| OCR 신뢰도 50~80% | ⚠️ 사용자 확인 | 값 표시 + "맞습니까?" 확인 요청 |
| OCR 신뢰도 80% 이상 | ✅ 자동 진행 | 바로 판정 실행 |
| 연속 3회 인식 실패 | ❌ 수동 입력 전환 | "수동으로 값을 입력하세요" 안내 |

### 촬영 품질 가이드 (사용자에게 표시)
- 화면이 뚜렷하게 보이도록 **정면에서** 촬영
- **반사/그림자**가 없는 각도에서 촬영
- 판정 영역이 화면의 **50% 이상**을 차지하도록 가까이 촬영
- 흔들림 없이 **안정적으로** 촬영 (0.5초 정지)

## 4. 파일럿 합격 기준

| 항목 | 기준 | 검증 방법 |
|------|------|----------|
| 설비 등록 | 3개 이상 설비, 5개 이상 QR 포인트 | 위치 탭 확인 |
| 기준설정 | 각 QR에 영역+조건 설정 완료 | 기준설정 탭 확인 |
| 판정 실행 | 동일 설비 10회 연속 판정 | 판정 탭 반복 테스트 |
| 판정 성공률 | 10회 중 9회 이상 정확 (90%+) | 결과 검증 |
| 오판정 수정 | 수정 → 재판정 시 정확도 향상 | 학습 효과 확인 |
| 이력 조회 | 판정 이력 10건 이상 조회 가능 | 결과 탭 확인 |
| 오프라인 동작 | 인터넷 없이 전체 기능 작동 | 네트워크 차단 후 테스트 |
